git push 할때마다 target폴더까지 업로드가 되서 pull 하면 무조건 충돌나는 귀찮은 상태가 발생하는데요.
맨처음 한번 add를 하면 git이 이 폴더나 파일을 계속 주시 합니다.
그렇기에 target폴더를 git 에서 제외 시켜야 되죠.
git bash프로그램을 열여서 unknownbike까지 이동시켜서..
git rm /target --cached 라는 명령어를 쳐줍니다(실제 폴더 삭제가 아니라 git에서 제외시키는 명령어)
git status 로 확인해보면 초록색으로 deleted가 됬을겁니다.
git commit -m "아무거나" 로 커밋까지 해주면 git 에서 제외가 되는 겁니다.
근대 뭔가 작업을 하면 또 git status에 target폴더가 빨간색으로 표시가 될텐데요..
git status에서도 표시 안되게 할려면 .gitignore라는 파일에 무시할 파일,폴더를 명시 해줘야 합니다.
unknownbike폴더에 .gitignore 라는 파일이 있는지 확인해 보고 없으면 github에서 받으세요 제가 target폴더 제외시키는 .gitignore 올려 놨습니다.
